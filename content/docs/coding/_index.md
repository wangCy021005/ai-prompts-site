# 💻 编程类提示词

RD 日常高频场景的 AI 提示词。来自真实开发实操，不是网上抄的空泛模板。

## 提示词列表

- [代码审查提示词](./代码审查/) — 让 AI 帮你审 PR，找 bug、提优化建议
- [Bug 定位提示词](./Bug定位/) — 描述现象让 AI 帮你定位 bug 根因
- [代码重构提示词](./代码重构/) — 让 AI 建议重构方案，提升可读性/性能
- [单元测试生成提示词](./单元测试/) — 根据代码让 AI 生成单元测试用例

## 日常研发流程

```
1. 写完代码  →  代码审查    →  提 PR 前自查
2. 线上报警  →  Bug 定位    →  快速定位根因
3. 接手老代码 →  代码重构    →  偷偷还技术债
4. 新功能完成 →  单元测试    →  补测试保平安
```

## 使用建议

1. 提示词里的方括号 `[xxx]` 是需要你替换的变量
2. 贴代码时只贴相关片段，别把整个文件糊上去，AI 会被噪声带偏
3. AI 给的方案一定要自己复核，它能给你思路，但责任你来背
4. 涉及线上 bug，先用 AI 给方向，最终还是要靠日志和断点确认

## 适用模型

- 复杂代码审查 / 重构方案：用 Claude Sonnet/Opus、GPT-4 级别模型
- 单测生成 / 简单 bug 定位：Haiku/GPT-4o-mini 也能跑，省 token
- 长上下文（整文件审查）：优先选 200K 上下文窗口的模型
