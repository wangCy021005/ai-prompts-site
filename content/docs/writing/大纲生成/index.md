# 大纲生成提示词

> **用途**：写小说前生成主角设定、金手指、章节大纲、单章细纲。
> **适用**：网文开新书前规划，重生系统流、赘婿逆袭、年代文等题材。

## 提示词 1：主角设定 + 金手指

开新书第一步，先把主角和金手指定下来。

```
你是一位资深网文策划，擅长设计高爽度系统流主角。

请帮我设计一个网文主角，要求：
- 题材：[重生系统流 / 赘婿逆袭 / 年代文，选一个]
- 主角性别：[男/女]
- 主角姓名：[自己起，或让 AI 建议 3 个]
- 金手指类型：[系统/重生记忆/异能/空间，选一个]

请输出：
1. 主角背景（200 字）：穿越前身份 + 穿越后处境
2. 金手指设定（300 字）：具体能力 + 使用限制 + 升级路径
3. 主角性格（100 字）：3 个核心性格特质
4. 核心爽点设计（200 字）：这本书靠什么爽？打脸？装逼？暴富？
5. 1 句话简介：用于签约
6. 50 字简介：用于书籍介绍页

要求：爽点密集、套路清晰、AI 容易续写。
```

## 提示词 2：前 30 章大纲

主角定好后，生成前 30 章大纲，明确每章爽点和钩子。

```
你是一位日更万字的网文作者，擅长[题材]。

我的小说设定如下：
[粘贴上面生成的主角设定]

请生成前 30 章的详细大纲，要求：
1. 每章 1-2 句话概括剧情
2. 每章标注 1 个爽点（打脸/收获/装逼/逆袭）
3. 每 5 章一个小高潮
4. 每 10 章一个大高潮
5. 章节结尾都要有钩子（留悬念）

输出格式：
第1章：[剧情] | 爽点：[xxx] | 钩子：[xxx]
第2章：...
```

## 提示词 3：单章细纲（写之前用）

每天写之前，把当章扩写成细纲，5 分钟搞定。

```
根据以下大纲，把第 [N] 章扩写为细纲：

[粘贴第 N 章的大纲]

请输出：
1. 本章 3 个场景（地点+人物+事件）
2. 每个场景的关键对话点
3. 本章爽点如何铺垫和爆发
4. 结尾钩子具体写法
5. 预计字数 4000 字，分配到 3 个场景
```

## 使用建议

1. 大纲生成为主，**人工必须过一遍**，改掉太离谱的情节
2. 细纲可以每天写之前现生成
3. 大纲存档，方便追溯（git 管理是好习惯）

## 大纲设计要点

| 要素 | 说明 |
|------|------|
| 爽点节奏 | 每 5 章小高潮，每 10 章大高潮 |
| 钩子 | 每章结尾留悬念，让读者追下一章 |
| 主角成长 | 金手指逐步升级，别一开始就满级 |
| 对手升级 | 校园对手 → 商业对手 → 资本对手 |

## 注意事项

⚠️ **不要照抄 AI 生成的大纲**，人工调整剧情走向，避免套路化。

⚠️ **大纲是活的**，写到中途可以根据数据调整，不必死守初期规划。
