# 小说续写提示词

> **用途**：日更网文续写，保持爽点节奏，对话占比 50%+，章末留钩子。
> **适用**：番茄/七猫等日更爽文，重生系统流、赘婿逆袭等题材。

## 提示词

```
你是一位日更万字的网文作者，写一本[题材]小说。

【小说设定】
主角：[姓名]，[一句话设定]
金手指：[一句话]
当前阶段：[第几章，主角处境]

【上一章结尾】
[粘贴上一章最后 500 字]

【本章细纲】
[粘贴本章细纲，或简单写本章要发生什么]

请续写本章正文，要求：
1. 字数 4000 字左右
2. 爽点密集：每 1000 字至少 1 个反转/打脸/收获
3. 对话占比 50% 以上，少环境描写和心理独白
4. 章节结尾留钩子，让读者想看下一章
5. 语言直白、节奏快，不要"文学腔"
6. 避免 AI 味词汇：在这个世界中、不禁、映入眼帘、仿佛、似乎、然而、不得不说
7. 人物说话要有个性，不要都是书面语

直接输出正文，不要解释。
```

## 使用说明

1. 把 `[题材]`、`[姓名]`、`[一句话设定]` 等方括号内容替换成你的设定
2. 上一章结尾粘贴 500 字左右，让 AI 接上剧情
3. 本章细纲可以简单写"本章主角打脸 XX，获得 XX"，不用太细

## 分场景续写（数据差时用）

如果整章生成质量不稳，拆成 3 个场景分别生成：

```
【场景 1】[场景描述，约 1200 字]
要求：开场抓人，前 200 字内必须出现冲突。
请写出这个场景的正文。

【场景 2】[场景描述，约 1500 字]
要求：承接场景 1，推进剧情，铺垫本章爽点。
请写出这个场景的正文。

【场景 3】[场景描述，约 1300 字]
要求：爽点爆发 + 章节结尾钩子。
请写出这个场景的正文。
```

## 开头专用（第 1 章特别重要）

第 1 章决定读者去留，单独用更精细的提示词：

```
你是一位网文大神，擅长写让人停不下来的开头。

我的小说设定：[粘贴主角设定]

请写第 1 章，3500-4000 字，要求：
1. 前 200 字内：主角穿越/重生 + 立刻遭遇危机
2. 前 1000 字内：金手指觉醒
3. 第 1 章必须有：1 个打脸 + 1 个收获 + 1 个悬念
4. 节奏极快，不要任何铺垫性描写
5. 结尾钩子：让读者必须点第 2 章

参考爆款开头节奏：开局即冲突，3 章一小爽，5 章一大爽。
```

## 调试技巧

| 问题 | 解决 |
|------|------|
| AI 写得太"温柔" | 加"语言要狠，反派要可恶，打脸要爽" |
| 对话太书面 | 加"角色说话用口语，可以带脏字和口头禅" |
| 节奏太慢 | 加"砍掉所有环境描写，只写动作和对话" |
| 字数不够 | 加"每个场景展开写，对话要来回几个回合" |

## 效果示例

用这个提示词生成的正文，应该具备以下特征：

- **开局即冲突**：前 200 字内出现矛盾或危机
- **对话占比高**：50% 以上，少环境描写
- **节奏快**：每 1000 字至少 1 个反转/打脸/收获
- **章末有钩子**：留悬念让读者追下一章

生成后通读一遍，确认这四点都达标，不达标就让 AI 重写或自己改。

## 注意事项

⚠️ **必须人工润色**：番茄等平台在升级 AI 检测，纯 AI 不润色直接发会被限流。生成后通读一遍，改几个词、加几句自己的话。

⚠️ **不要照抄设定**：提示词里的主角名、设定只是示例，换成你自己的。
